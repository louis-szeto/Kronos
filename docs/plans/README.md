# Plans Directory

This directory holds implementation plans for the Kronos project.

## Purpose

Plans are written **before** implementation begins. They describe:
- What will change
- Why it's changing
- Step-by-step approach
- Risk assessment

## Format

Each plan file should follow this structure:

```markdown
# [Plan Title]

## Problem
[What needs to change and why]

## Approach
[Chosen solution and rationale]

## Steps
1. [Specific action with file:line reference]
2. [Specific action]
...

## Risks
- [Potential issue and mitigation]

## Done Criteria
- [Measurable completion conditions]
```

## Naming Convention

- Use descriptive kebab-case names: `add-batch-prediction.md`, `refine-ddp-training.md`
- Include date prefix for time-bound plans: `2025-03-30-add-batch-prediction.md`
- For rejected alternatives: `rejected/` subdirectory
